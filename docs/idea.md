Идея микросервиса: сервис авторизуется через OAuth 2.0, далее получает последний отчет по специальному URL, затем по идентификатору этого отчета запрашивает список ошибочных объявлений и сохраняет их в базу данных MySQL. В системе может быть несколько личных кабинетов — для каждого из них необходимо аналогичным образом парсить отчеты и ошибочные объявления. Технические детали пока не уточняются.
